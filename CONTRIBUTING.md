## 🚀 How Can I Contribute?

### 1. Reporting Bugs
Bugs are tracked as GitHub issues. When creating an issue, please use the `Bug Report` template. Explain the problem and include additional details to help maintainers reproduce the problem:
* **Use a clear and descriptive title** for the issue to identify the problem.
* **Describe the exact steps which reproduce the problem** in as many details as possible.
* **Provide specific examples to demonstrate the steps.** 
* **Describe the behavior you observed after following the steps** and point out what exactly is the problem with that behavior.
* **Explain which behavior you expected to see instead and why.**

### 2. Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When creating an issue, please use the `Feature Request` template. Provide the following information:
* **Use a clear and descriptive title** for the issue to identify the suggestion.
* **Provide a step-by-step description of the suggested enhancement** in as many details as possible.
* **Provide specific examples to demonstrate the steps.**
* **Describe the current behavior** and **explain which behavior you expected to see instead**.
* **Explain why this enhancement would be useful** to most users.

### 3. Pull Requests
* Fill in the provided Pull Request template.
* Describe precisely what your pull request implements or fixes.
* Follow the styleguides and linting rules enforced in the codebase (`eslint`, `prettier`).
* End all files with a newline.

## 💻 Environment Setup
1. Fork the repo and create your branch from `main`.
2. Note: You must contact **saiyam468@gmail.com** for access to private `.env` keys if necessary for fundamental testing.
3. Run `npm install` to install dependencies.
4. Ensure your code passes all linting tests `npm run lint`.
5. Run the dev server `npm run dev` to ensure it integrates seamlessly.

## 🎨 Styleguides

### Git Commit Messages
* Use the present tense ("Add feature" not "Added feature").
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
* Limit the first line to 72 characters or less.
* Reference issues and pull requests liberally after the first line.

### JavaScript/TypeScript Styleguide
* All code is parsed through ESLint. Ensure there are no warnings or errors.
* We strongly encourage the usage of TypeScript `interface` / `type` to maintain strict types everywhere.
* **No `any` types allowed**. We enforce strict Next.js and React typing.
